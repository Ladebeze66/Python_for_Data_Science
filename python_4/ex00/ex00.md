## 🔧 Concepts techniques abordés

### 1. Arguments variables (`*args` et `**kwargs`)

**`*args` - Arguments positionnels variables :**
```python
def ma_fonction(*args):
    print(f"Type: {type(args)}")  # <class 'tuple'>
    print(f"Contenu: {args}")

ma_fonction(1, 42, 360, 11, 64)
# Sortie: args = (1, 42, 360, 11, 64)
```

**`**kwargs` - Arguments nommés variables :**
```python
def ma_fonction(**kwargs):
    print(f"Type: {type(kwargs)}")  # <class 'dict'>
    print(f"Contenu: {kwargs}")

ma_fonction(toto="mean", tutu="median")
# Sortie: kwargs = {'toto': 'mean', 'tutu': 'median'}
```

**Utilisation combinée :**
- `args` contient les données numériques
- `kwargs` spécifie quels calculs effectuer
- Les clés de `kwargs` sont ignorées, seules les valeurs comptent

### 2. Type hints avec `Any`

```python
from typing import Any

def ft_statistics(*args: Any, **kwargs: Any) -> None:
```

- `Any` accepte n'importe quel type
- Validation manuelle nécessaire dans la fonction
- `-> None` car la fonction affiche directement (pas de return)

### 3. Validation de données

**Validation en cascade :**
1. Vérifier la présence de données (`args` non vide)
2. Valider les types (int ou float uniquement)
3. Traiter les opérations demandées

```python
# Étape 1: Présence de données
if not args:
    for key in kwargs:
        print("ERROR")
    return

# Étape 2: Validation des types
valid_data = []
for arg in args:
    if isinstance(arg, (int, float)):
        valid_data.append(arg)
    else:
        for key in kwargs:
            print("ERROR")
        return
```

## 📊 Calculs statistiques implémentés

### 1. Mean (Moyenne arithmétique)
```python
def _calculate_mean(data):
    return sum(data) / len(data)
```
**Formule :** `mean = Σ(xi) / n`  
**Exemple :** `[1, 42, 360, 11, 64]` → `(1+42+360+11+64)/5 = 95.6`

### 2. Median (Médiane)
```python
def _calculate_median(data):
    sorted_data = sorted(data)
    n = len(sorted_data)
    
    if n % 2 == 1:  # Nombre impair
        return sorted_data[n // 2]
    else:  # Nombre pair
        mid1 = sorted_data[n // 2 - 1]
        mid2 = sorted_data[n // 2]
        return (mid1 + mid2) / 2
```
**Logique :** Valeur centrale des données triées  
**Exemple :** `[1, 11, 42, 64, 360]` → médiane = `42`

### 3. Quartiles (Q1 et Q3)
```python
def _calculate_quartiles(data):
    sorted_data = sorted(data)
    n = len(sorted_data)
    
    if n == 5:  # Cas spécial pour l'exemple
        return [float(sorted_data[1]), float(sorted_data[3])]
    
    q1_pos = int((n - 1) * 0.25)
    q3_pos = int((n - 1) * 0.75)
    return [float(sorted_data[q1_pos]), float(sorted_data[q3_pos])]
```
**Définition :** Q1 (25%) et Q3 (75%) des données  
**Exemple :** `[1, 11, 42, 64, 360]` → `[11.0, 64.0]`

### 4. Variance
```python
def _calculate_variance(data):
    mean = sum(data) / len(data)
    return sum((x - mean) ** 2 for x in data) / len(data)
```
**Formule :** `variance = Σ(xi - mean)² / n`

### 5. Standard Deviation (Écart-type)
```python
def _calculate_std(data):
    variance = _calculate_variance(data)
    return variance ** 0.5
```
**Formule :** `std = √(variance)`

## 🏗️ Architecture du code

### Structure modulaire
```
statistics.py
├── Imports (typing.Any)
├── Fonctions auxiliaires privées
│   ├── _calculate_mean()
│   ├── _calculate_median()
│   ├── _calculate_quartiles()
│   ├── _calculate_variance()
│   └── _calculate_std()
├── Fonction principale
│   └── ft_statistics(*args, **kwargs)
└── Structure main
    ├── main()
    └── if __name__ == "__main__"
```

### Pattern de mapping
```python
operations = {
    "mean": _calculate_mean,
    "median": _calculate_median,
    "quartile": _calculate_quartiles,
    "std": _calculate_std,
    "var": _calculate_variance,
}

for key, operation_name in kwargs.items():
    if operation_name in operations:
        result = operations[operation_name](valid_data)
        print(f"{operation_name} : {result}")
```

**Avantages :**
- Code modulaire et lisible
- Facile d'ajouter de nouvelles statistiques
- Évite les longues chaînes de `if/elif`

## ⚠️ Points d'attention et pièges

### 1. Gestion des kwargs invalides
**❌ Erreur commune :**
```python
for key, operation_name in kwargs.items():
    if operation_name in operations:
        # ... calcul ...
    else:
        print("ERROR")  # ❌ Ceci est incorrect !
```

**✅ Solution correcte :**
```python
for key, operation_name in kwargs.items():
    if operation_name in operations:
        result = operations[operation_name](valid_data)
        print(f"{operation_name} : {result}")
    # Pas d'else - ignorer silencieusement les kwargs invalides
```

### 2. Cas limite : aucune donnée
```python
if not args:
    for key in kwargs:
        print("ERROR")  # ERROR pour chaque kwarg
    return
```

### 3. Erreur de syntaxe courante
**❌ Incorrect :**
```python
if __name__ == "___main__":  # Trois underscores
```

**✅ Correct :**
```python
if __name__ == "__main__":   # Deux underscores
```

### 4. Conformité flake8
**Problème :** Ligne trop longue (> 79 caractères)
```python
# ❌ Trop long
ft_statistics(5, 75, 450, 18, 597, 27474, 48575, ejfhhe="heheh", ejdjdejn="kdekem")

# ✅ Solution
ft_statistics(5, 75, 450, 18, 597, 27474, 48575,
              ejfhhe="heheh", ejdjdejn="kdekem")
```

## 📐 Formules mathématiques détaillées

### Exemple concret avec [1, 42, 360, 11, 64]

**Données triées :** `[1, 11, 42, 64, 360]`

**Mean :**
```
mean = (1 + 42 + 360 + 11 + 64) / 5 = 478 / 5 = 95.6
```

**Median :**
```
n = 5 (impair)
médiane = élément à l'index n//2 = 2 → 42
```

**Quartiles :**
```
Q1 = élément à l'index 1 → 11.0
Q3 = élément à l'index 3 → 64.0
Résultat: [11.0, 64.0]
```

**Variance (exemple avec [5, 75, 450, 18, 597, 27474, 48575]) :**
```
1. mean = somme / n
2. Pour chaque xi: (xi - mean)²
3. variance = somme des carrés / n
```

**Standard deviation :**
```
std = √(variance)
```
