## 🔍 Phase 1 : Analyse et compréhension

### 1. **Analyse du prototype et des contraintes**

Le prototype imposé est une structure à **3 niveaux** :

**Niveau 1 - `callLimit(limit: int)` :**
- `limit: int` : Nombre maximum d'appels autorisés
- `count = 0` : Compteur d'appels (fourni dans le prototype)
- **Rôle :** Factory qui crée des décorateurs personnalisés

**Niveau 2 - `callLimiter(function)` :**
- `function` : La fonction à décorer (f ou g dans l'exemple)
- **Rôle :** Décorateur proprement dit qui reçoit la fonction

**Niveau 3 - `limit_function(*args, **kwds)` :**
- `*args, **kwds` : Arguments de la fonction originale
- **Rôle :** Wrapper qui remplace la fonction originale

**Contraintes importantes :**
- ✅ Aucune fonction externe autorisée
- ✅ Variables globales interdites
- ✅ Structure main() obligatoire
- ✅ Docstrings obligatoires

### 2. **Compréhension des décorateurs Python**

**Qu'est-ce qu'un décorateur ?**
Un décorateur est une fonction qui modifie ou étend le comportement d'une autre fonction sans la modifier directement.

**Syntaxe de base :**
```python
# Sans paramètre
@mon_decorateur
def ma_fonction():
    pass

# Équivalent à :
ma_fonction = mon_decorateur(ma_fonction)
```

**Avec paramètres (notre cas) :**
```python
# Avec paramètre
@callLimit(3)
def f():
    print("f()")

# Équivalent à :
f = callLimit(3)(f)
# Étapes :
# 1. callLimit(3) retourne callLimiter
# 2. callLimiter(f) retourne limit_function
# 3. f = limit_function
```

### 3. **Analyse des exemples d'entrée/sortie**

**Code de test :**
```python
from callLimit import callLimit

@callLimit(3)
def f():
    print("f()")

@callLimit(1)
def g():
    print("g()")

for i in range(3):
    f()
    g()
```

**Analyse détaillée :**

**Fonction `f` avec limite 3 :**
- **1er appel :** `f()` → `"f()"` (1/3 appels)
- **2ème appel :** `f()` → `"f()"` (2/3 appels)
- **3ème appel :** `f()` → `"f()"` (3/3 appels)

**Fonction `g` avec limite 1 :**
- **1er appel :** `g()` → `"g()"` (1/1 appel)
- **2ème appel :** `g()` → `"Error: <function g at 0x...> call too many times"`
- **3ème appel :** `g()` → `"Error: <function g at 0x...> call too many times"`

**Sortie attendue :**
```

Error: <function g at 0x7fabdc243ee0> call too many times
```

### 4. **Logique de limitation d'appels**

**Mécanisme détaillé :**

**Étape 1 - Création du décorateur :**
```python
# @callLimit(3) sur f()
decorator_for_f = callLimit(3)  # Crée un décorateur avec limit=3, count=0
f = decorator_for_f(f)          # Applique le décorateur à f
```

**Étape 2 - Premier appel de f() :**
```python
# f() est maintenant limit_function
count = 0 → 1  # Incrémente le compteur
if count <= limit:  # 1 <= 3 → True
    return original_f(*args, **kwds)  # Exécute la fonction originale
```

**Étape 3 - Appels suivants :**
```python
# Deuxième appel : count = 1 → 2, 2 <= 3 → OK
# Troisième appel : count = 2 → 3, 3 <= 3 → OK  
# Quatrième appel : count = 3 → 4, 4 <= 3 → False → Error
```

### 5. **Structure à 3 niveaux expliquée**

**Pourquoi 3 niveaux ?**

**Niveau 1 - Configuration :**
```python

    # Retourne un décorateur configuré
```

**Niveau 2 - Décoration :**
```python
def callLimiter(function):  # Reçoit la fonction à décorer
    # Retourne le wrapper qui remplacera la fonction
```

**Niveau 3 - Exécution :**
```python
def limit_function(*args, **kwds):  # Remplace la fonction originale
    # Logique de limitation + appel de la fonction originale
```

## 🔧 Phase 2 : Concepts théoriques

### 1. **Décorateurs avec paramètres (decorator factory)**

**Notre cas - Structure à 3 niveaux :**
```python
def callLimit(limit: int):           # Niveau 1: Factory
    count = 0                        # État partagé
    
    def callLimiter(function):       # Niveau 2: Décorateur
        def limit_function(*args, **kwds):  # Niveau 3: Wrapper
            # Logique de limitation
            pass
        return limit_function
    return callLimiter

# Utilisation :
@callLimit(3)  # callLimit(3) retourne callLimiter
def f():       # callLimiter(f) retourne limit_function
    pass       # f = limit_function
```

**Décomposition étape par étape :**
```python
# @callLimit(3) équivaut à :
step1 = callLimit(3)        # step1 = callLimiter (avec limit=3, count=0)
step2 = step1(f)           # step2 = limit_function (avec access à f, limit, count)
f = step2                  # f est maintenant limit_function
```

### 2. **Closures avancées pour maintenir l'état**

**Comparaison avec l'exercice 01 :**

**ex01 - Closure simple :**
```python
def outer(x, function):
    count = 0
    def inner():
        nonlocal count, x
        count += 1
        x = function(x)
        return x
    return inner
```

**ex02 - Closure avec décorateur :**
```python
def callLimit(limit):
    count = 0  # État partagé par toutes les instances du décorateur
    def callLimiter(function):
        def limit_function(*args, **kwds):
            nonlocal count  # Accès à l'état du niveau supérieur
            count += 1
            # Logique de limitation
        return limit_function
    return callLimiter
```

**Différence clé :**
- **ex01 :** Une closure par appel à `outer()`
- **ex02 :** Une closure par fonction décorée, mais état partagé par décorateur

### 3. **`*args` et `**kwargs` dans les wrappers**

**Pourquoi `*args` et `**kwds` ?**
Le wrapper doit pouvoir accepter n'importe quelle signature de fonction :

```python
@callLimit(2)
def func_no_args():
    pass

@callLimit(2)  
def func_with_args(a, b, c):
    pass

@callLimit(2)
def func_with_kwargs(name="test", value=42):
    pass

@callLimit(2)
def func_mixed(a, b, *args, **kwargs):
    pass
```

**Le wrapper universel :**
```python
def limit_function(*args, **kwds):
    # Peut accepter toutes les signatures ci-dessus
    if count <= limit:
        return function(*args, **kwds)  # Transmet tous les arguments
```

### 4. **Gestion d'erreurs personnalisées**

**Format du message d'erreur :**
```python
def limit_function(*args, **kwds):
    nonlocal count
    count += 1
    
    if count <= limit:
        return function(*args, **kwds)
    else:
        print(f"Error: {function} call too many times")
```

**Analyse du format :**
- `{function}` → Utilise la représentation par défaut de l'objet fonction
- `<function g at 0x7fabdc243ee0>` → Format automatique de Python

### 5. **Pattern de décorateur avec état**

**Structure complète :**
```python
def callLimit(limit: int):
    """Factory de décorateurs avec limitation d'appels"""
    
    def callLimiter(function):
        """Décorateur qui limite les appels d'une fonction"""
        count = 0  # Compteur spécifique à cette fonction
        
        def limit_function(*args, **kwds):
            """Wrapper qui contrôle les appels"""
            nonlocal count
            count += 1
            
            if count <= limit:
                return function(*args, **kwds)
            else:
                print(f"Error: {function} call too many times")
                
        return limit_function
    return callLimiter
```

**Attention - Placement du compteur :**
```python
# ✅ Bon placement selon le prototype (un compteur par callLimit)
def callLimit(limit: int):
    count = 0  # Spécifique à cette instance de callLimit
    def callLimiter(function):
        def limit_function(*args, **kwds):
            # ...
```

### 6. **Comparaison avec d'autres patterns**

**Décorateur vs Classe :**
```python
# Approche décorateur (notre exercice)
@callLimit(3)
def f():
    pass

# Approche classe (alternative)
class CallLimiter:
    def __init__(self, limit):
        self.limit = limit
        self.count = 0
    
    def __call__(self, *args, **kwargs):
        self.count += 1
        if self.count <= self.limit:
            return self.original_func(*args, **kwargs)
        else:
            print(f"Error: {self.original_func} call too many times")
```

### 3. **Fonction `limit_function(*args, **kwds)` - wrapper final**

```python
def limit_function(*args: Any, **kwds: Any):
    """
    Wrapper qui remplace la fonction originale et contrôle les appels.
    
    Args:
        *args: Arguments positionnels de la fonction originale
        **kwds: Arguments nommés de la fonction originale
        
    Returns:
        Résultat de la fonction originale ou None si limitée
    """
    nonlocal count  # Accès au compteur du scope parent
    
    count += 1  # Incrémente le compteur à chaque appel
    
    if count <= limit:
        # Appel autorisé - exécute la fonction originale
        return function(*args, **kwds)
    else:
        # Limite dépassée - affiche l'erreur
        print(f"Error: {function} call too many times")
```

```python
            """Wrapper qui contrôle les appels"""
            nonlocal count
            count += 1

            if count <= limit:
                return function(*args, **kwargs)
            else:
                print(f"Error: {function} call too many times")

        return limit_function
    return callLimiter
```
