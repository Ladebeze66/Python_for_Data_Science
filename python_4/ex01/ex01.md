## 🔍 Phase 1 : Analyse et compréhension

### 1. **Analyse du prototype et des contraintes**

**Fonction `square()` :**
- `x: int | float` : Accepte un nombre (entier ou flottant)
- `-> int | float` : Retourne un nombre
- **Objectif :** Calculer le carré de x (x²)

**Fonction `pow()` :**

- `-> int | float` : Retourne un nombre  
- **Objectif :** Calculer x^x (exponentiation par soi-même)

**Fonction `outer()` :**
- `x: int | float` : Valeur initiale
- `function` : Une fonction à appliquer (square ou pow)
- `-> object` : Retourne un objet (la fonction inner)
- `count = 0` : Variable de comptage (fournie dans le prototype)
- `inner() -> float` : Fonction imbriquée qui fait le travail

### 2. **Compréhension des concepts : closures et fonctions imbriquées**

**Qu'est-ce qu'une closure ?**
Une closure est une fonction qui "capture" des variables de son environnement lexical et les conserve même après que la fonction parent ait terminé.

**Exemple simple :**
```python
def make_counter():
    count = 0  # Variable locale
    
    def increment():
        nonlocal count  # Accès à la variable du scope parent
        count += 1
        return count
    
    return increment  # Retourne la fonction inner

# Utilisation
counter = make_counter()  # counter est maintenant la fonction increment
print(counter())  # 1
print(counter())  # 2
print(counter())  # 3
```

### 3. **Analyse des exemples d'entrée/sortie**

**Exemple 1 - avec square :**
```python
my_counter = outer(3, square)
print(my_counter())  # 9
print(my_counter())  # 81  
print(my_counter())  # 6561
```

**Logique de fonctionnement :**
- `outer(3, square)` crée une closure avec valeur initiale 3 et fonction square
- **1er appel :** `3² = 9` (count=1)
- **2ème appel :** `9² = 81` (count=2) 
- **3ème appel :** `81² = 6561` (count=3)

**Exemple 2 - avec pow :**
```python
another_counter = outer(1.5, pow)
print(another_counter())  # 1.8371173070873836
print(another_counter())  # 3.056683336818703
print(another_counter())  # 30.42684786675409
```

**Logique de fonctionnement :**
- `outer(1.5, pow)` crée une closure avec valeur initiale 1.5 et fonction pow
- **1er appel :** `1.5^1.5 ≈ 1.837` (count=1)
- **2ème appel :** `1.837^1.837 ≈ 3.057` (count=2)
- **3ème appel :** `3.057^3.057 ≈ 30.427` (count=3)

### 4. **Logique de fonctionnement progressive**

**Mécanisme détaillé :**

1. **Initialisation :** `outer(x, function)` crée une closure
   - `x` = valeur de départ
   - `function` = transformation à appliquer
   - `count` = compteur d'appels

2. **Premier appel de inner() :**
   - `count` passe de 0 à 1
   - Applique `function(x)` → nouveau résultat
   - Stocke le résultat pour le prochain appel

3. **Appels suivants :**
   - `count` s'incrémente
   - Applique `function(résultat_précédent)`
   - Continue la chaîne de transformations

## 🔧 Phase 2 : Concepts théoriques

### 1. **Closures (fermetures) en Python**

**Définition :**
Une closure est une fonction qui "ferme" sur des variables de son environnement lexical, les conservant même après que la fonction parent ait terminé son exécution.

**Mécanisme dans notre exercice :**
```python
def outer(x, function):
    count = 0  # Variables capturées par la closure
    
    def inner():
        # inner() a accès à x, function, et count
        # même après que outer() ait terminé
        nonlocal count  # Permet de modifier count
        count += 1
        # ... logique de calcul
    
    return inner  # Retourne la closure
```

### 2. **Fonctions imbriquées et portée des variables**

**Règle LEGB (Local, Enclosing, Global, Built-in) :**
```python
global_var = "global"

def outer_function():
    enclosing_var = "enclosing"
    
    def inner_function():
        local_var = "local"
        
        # Accès aux différents scopes
        print(local_var)      # Local
        print(enclosing_var)  # Enclosing (closure)
        print(global_var)     # Global
        print(len)            # Built-in
    
    return inner_function
```

**Mot-clé `nonlocal` :**
```python
def outer():
    count = 0  # Variable du scope enclosing
    
    def inner():
        nonlocal count  # Permet de modifier count
        count += 1      # Sans nonlocal, erreur !
        return count
    
    return inner

counter = outer()
print(counter())  # 1
print(counter())  # 2
```

### 3. **Fonctions d'ordre supérieur (Higher-order functions)**

**Définition :** Fonctions qui prennent d'autres fonctions comme arguments ou retournent des fonctions.

**Notre cas :**
```python
def outer(x, function):  # Prend 'function' comme paramètre
    # ...
    def inner():
        result = function(current_value)  # Utilise la fonction passée
        # ...
    return inner  # Retourne une fonction
```

### 4. **État persistant avec les closures**

**Problème à résoudre :** Comment maintenir une valeur entre plusieurs appels de fonction ?

**Solution avec closure (✅ CORRECTE) :**
```python
def make_stateful_function(initial_value):
    current_value = initial_value  # État encapsulé
    
    def stateful_function():
        nonlocal current_value
        current_value = some_calculation(current_value)
        return current_value
    
    return stateful_function
```

### 5. **Type hints avec union `int | float`**

**Syntaxe moderne (Python 3.10+) :**
```python
def square(x: int | float) -> int | float:
    return x * x

def pow(x: int | float) -> int | float:
    return x ** x
```

**Pourquoi cette union ?**
- Les calculs mathématiques acceptent entiers et flottants
- Le résultat peut être int ou float selon l'entrée
- `3² = 9` (int) mais `1.5^1.5 ≈ 1.837` (float)

### 6. **Pattern Factory avec closures**

**Notre exercice utilise le pattern Factory :**
```python
def outer(x, function):  # Factory function
    # Configuration initiale
    count = 0
    current_value = x
    
    def inner():  # Produit de la factory
        # Logique métier
        nonlocal count, current_value
        count += 1
        current_value = function(current_value)
        return current_value
    
    return inner  # Retourne le "produit"

# Création de différents "produits"
square_processor = outer(2, square)
pow_processor = outer(1.5, pow)
```

## 🏗️ Phase 3 : Structure du code

### 1. **Fonction `square()` - calcul du carré**

```python
def square(x: int | float) -> int | float:
    """
    Calcule le carré d'un nombre.
    
    Args:
        x: Nombre à élever au carré
        
    Returns:
        Le carré de x (x²)
    """
    return x * x
```

### 2. **Fonction `pow()` - exponentiation par soi-même**

```python
def pow(x: int | float) -> int | float:
    """
    Calcule l'exponentiation d'un nombre par lui-même.
    
    Args:
        x: Nombre à élever à sa propre puissance
        
    Returns:
        x élevé à la puissance x (x^x)
    """
    return x ** x
```

### 3. **Structure de base de `outer()`**

```python
def outer(x: int | float, function) -> object:
    """
    Crée une closure qui applique successivement une fonction.
    
    Args:
        x: Valeur initiale
        function: Fonction à appliquer (square ou pow)
        
    Returns:
        Une fonction (closure) qui maintient l'état entre les appels
    """
    count = 0  # Compteur d'appels (fourni dans le prototype)
    
    def inner() -> float:
        """Fonction interne qui effectue les calculs successifs"""
        nonlocal count, x
        count += 1
        x = function(x)
        return x
    
    return inner  # Retourne la closure
```

### **Étape 2 : Implémentation de la logique `inner()`**

```python
def inner() -> float:
    """
    Applique la fonction sur la valeur courante et met à jour l'état.
    
    Returns:
        Résultat du calcul après application de la fonction
    """
    nonlocal count, x  # Accès aux variables du scope enclosing
    
    count += 1         # Incrémente le compteur d'appels
    x = function(x)    # Applique la fonction sur la valeur courante
    return x           # Retourne le nouveau résultat
```

**Explication ligne par ligne :**
1. `nonlocal count, x` : Permet de modifier les variables du scope parent
2. `count += 1` : Incrémente le compteur (peut servir pour debugging)
3. `x = function(x)` : **Cœur de la logique** - applique la transformation
4. `return x` : Retourne le résultat et le garde pour le prochain appel

## 🔍 Points techniques importants

### **1. Pourquoi `nonlocal` est nécessaire**

```python
def outer(x, function):
    count = 0
    
    def inner():
        # Sans nonlocal
        count = count + 1  # ❌ UnboundLocalError!
        
        # Avec nonlocal  
        nonlocal count
        count += 1  # ✅ Fonctionne
```

### **2. Différence avec les variables globales**

```python
# ❌ Approche interdite (variables globales)
current_value = 0
def bad_function():
    global current_value
    current_value = square(current_value)
    return current_value

# ✅ Approche correcte (closure)
def outer(x, function):
    def inner():
        nonlocal x
        x = function(x)
        return x
    return inner
```

### **3. Type de retour `-> object` pour `outer()`**

- `outer()` retourne une fonction (`inner`)
- En Python, les fonctions sont des objets
- `-> object` est le type le plus générique pour une fonction
