### 2. **Compréhension des dataclasses Python**

**Qu'est-ce qu'une dataclass ?**
Une dataclass est une classe Python qui génère automatiquement des méthodes communes basées sur les annotations de type des attributs.

**Exemple simple :**
```python
@dataclass
class Person:
    name: str
    age: int

# Équivalent à :
class Person:
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age
    
    def __repr__(self):
        return f"Person(name='{self.name}', age={self.age})"
    
    def __eq__(self, other):
        return (self.name, self.age) == (other.name, other.age)
```

### 4. **Logique de génération automatique des champs**

**Génération du login :**
```python
# name="Edward", surname="agle" → login="Eagle"
```

**Logique probable :**
1. Prendre la première lettre de `name` : "E"
2. Prendre le reste de `surname` : "agle"
3. Concaténer : "E" + "agle" = "Eagle"
4. Mettre en forme (première lettre majuscule) : "Eagle"

**Génération de l'ID :**
```python
def generate_id() -> str:
    return "".join(random.choices(string.ascii_lowercase, k=15))
```

**Logique :**
- `string.ascii_lowercase` : "abcdefghijklmnopqrstuvwxyz"
- `random.choices(..., k=15)` : 15 caractères aléatoires avec remise
- `"".join()` : Concatène en une chaîne

### 5. **Structure des champs de la classe**

**Champs requis (initialisables) :**
- `name: str` : Nom de l'étudiant
- `surname: str` : Prénom de l'étudiant

**Champs optionnels (initialisables) :**
- `active: bool = True` : Statut actif par défaut

**Champs calculés (non initialisables) :**
- `login: str` : Généré à partir de name + surname
- `id: str` : Généré aléatoirement

### 6. **Utilisation de `field()` pour la configuration**

**Pourquoi `field()` ?**
Pour configurer le comportement des champs :
- `init=False` : Le champ n'apparaît pas dans `__init__()`
- `default_factory` : Fonction appelée pour générer la valeur par défaut

**Exemple d'utilisation :**
```python
@dataclass
class Student:
    name: str
    surname: str
    active: bool = True
    login: str = field(init=False)  # Non initialisable
    id: str = field(init=False, default_factory=generate_id)  # Généré automatiquement
```

## 🔧 Phase 2 : Concepts théoriques

### 1. **Dataclasses en Python - décorateur `@dataclass`**

**Fonctionnement du décorateur :**
```python
@dataclass
class Student:
    name: str
    age: int

# Le décorateur génère automatiquement :
# - __init__(self, name: str, age: int)
# - __repr__(self) -> str
# - __eq__(self, other) -> bool
# - __hash__(self) (si frozen=True)
```

**Méthodes générées automatiquement :**

**`__init__()` :**
```python
def __init__(self, name: str, age: int):
    self.name = name
    self.age = age
```

**`__repr__()` :**
```python
def __repr__(self):
    return f"Student(name='{self.name}', age={self.age})"
```

### 2. **Champs avec `field()` - configuration avancée**

**Paramètres principaux de `field()` :**

**`init=False` :**
```python
@dataclass
class Student:
    name: str
    login: str = field(init=False)  # N'apparaît pas dans __init__()

# Utilisation :
student = Student(name="Alice")  # OK
student = Student(name="Alice", login="test")  # ❌ TypeError
```

**`default` vs `default_factory` :**
```python
@dataclass
class Student:
    # ❌ Problème : même liste partagée entre toutes les instances
    grades: list = field(default=[])
    
    # ✅ Correct : nouvelle liste pour chaque instance
    grades: list = field(default_factory=list)
    
    # ✅ Fonction personnalisée
    id: str = field(default_factory=generate_id)
```

### 3. **Génération automatique de champs calculés**

**Solution avec `__post_init__()` :**
```python
@dataclass
class Student:
    name: str
    surname: str
    login: str = field(init=False)
    
    def __post_init__(self):
        self.login = (self.name[0] + self.surname).capitalize()
```

**Ordre d'exécution :**
1. `__init__()` : Initialise les champs avec `init=True`
2. `default_factory` : Génère les champs avec `default_factory`
3. `__post_init__()` : Initialise les champs calculés
4. Instance prête à l'utilisation

### 4. **Restriction d'initialisation avec `init=False`**

**Mécanisme :**

### 5. **Fonctions de génération (`generate_id()`)**

**Analyse de la fonction fournie :**
```python
def generate_id() -> str:
    return "".join(random.choices(string.ascii_lowercase, k=15))
```

**Décomposition :**
- `string.ascii_lowercase` : `"abcdefghijklmnopqrstuvwxyz"`
- `random.choices(population, k=15)` : 15 éléments aléatoires avec remise
- `"".join(list)` : Concatène la liste en chaîne

**Différence avec `random.choice()` :**
```python
# random.choice() : un seul élément
char = random.choice(string.ascii_lowercase)  # 'a'

# random.choices() : plusieurs éléments
chars = random.choices(string.ascii_lowercase, k=5)  # ['a', 'b', 'c', 'd', 'e']
```

### 6. **Manipulation de chaînes et formatage**

**Génération du login - Analyse détaillée :**

**Exemple :** `name="Edward"`, `surname="agle"` → `login="Eagle"`

**Logique :**
```python
def generate_login(name: str, surname: str) -> str:
    return (name[0] + surname).capitalize()
```

### 7. **Ordre des champs et contraintes**

**Règle fondamentale :** Champs sans défaut avant champs avec défaut.

**✅ Correct :**

### 8. **Comparaison avec les classes traditionnelles**

**Classe traditionnelle :**
```python
class Student:
    def __init__(self, name: str, surname: str, active: bool = True):

        self.surname = surname
        self.active = active
        self.login = (name[0] + surname).capitalize()
        self.id = generate_id()

```

**Dataclass équivalente :**
```python
@dataclass
class Student:

    surname: str
    active: bool = True
    login: str = field(init=False)
    id: str = field(init=False, default_factory=generate_id)
    
    def __post_init__(self):
        self.login = (self.name[0] + self.surname).capitalize()
```

**Avantages des dataclasses :**
- ✅ Moins de code boilerplate
- ✅ Génération automatique des méthodes
- ✅ Type hints intégrés
- ✅ Configuration flexible avec `field()`

```python
import random
import string
```
```python
from dataclasses import dataclass, field
```


```python
def generate_id() -> str:
```
```python
    """
```
```python
    Génère un identifiant aléatoire de 15 caractères.

    Returns:
        str: Chaîne de 15 caractères alphabétiques minuscules
    """
    return "".join(random.choices(string.ascii_lowercase, k=15))


@dataclass
class Student:
    """
    Classe représentant un étudiant avec génération automatique de login et ID.

    Attributes:
        name: Nom de l'étudiant
        surname: Prénom de l'étudiant
        active: Statut actif (True par défaut)
        login: Login généré automatiquement (non initialisable)
        id: Identifiant unique généré automatiquement (non initialisable)
    """
    name: str
    surname: str
    active: bool = True
    login: str = field(init=False)
    id: str = field(init=False, default_factory=generate_id)

    def __post_init__(self):
        """
        Initialise les champs calculés après la création de l'instance.

        Génère le login à partir du nom et prénom selon la logique :
        première lettre du nom + prénom, le tout capitalisé.
        """
        self.login = (self.name[0] + self.surname).capitalize()


def main():
    """Fonction principale - tests dans tester.py"""
    pass


if __name__ == "__main__":
    main()
```
